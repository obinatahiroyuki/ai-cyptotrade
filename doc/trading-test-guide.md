# 取引テストガイド

本番移行前に、ペーパートレードモードで取引フローを検証する手順です。

---

## ペーパートレードモードとは

- **実際の注文は行わない**（Bitget API の買い/売りを呼ばない）
- シグナル解析・価格チェック・ポジション作成・価格監視・手動利確の**全フローをシミュレート**
- ダッシュボードの表示やDBへの記録は本番と同様に動作

---

## セットアップ

### 1. 環境変数に PAPER_TRADING を追加

`.env.local` に以下を追加:

```env
PAPER_TRADING=true
```

### 2. 必要な設定

- **Turso DB**: マイグレーション済みであること
- **Bitget API キー**: 取引所連携で登録（ペーパー時はAPIは呼ばないが、接続テスト用に必要）
- **Discord Bot**: シグナル監視をテストする場合に設定（`DISCORD_CHANNEL_IDS` はカンマ区切りで最大10件）
- **投資設定**: 設定 → 投資設定 で **自動売買 ON** にしておく

---

## テスト手順

### パターン A: Discord シグナル連動のテスト

1. **ワーカーを起動**
   ```bash
   npm run worker
   ```
   `[PAPER] PAPER TRADING MODE` と表示されればOK

2. **Discord にテストシグナルを投稿**
   
   監視チャンネルに以下のようなメッセージを投稿:
   ```
   🔶売買報告
   銘柄: BTC
   参考価格: 95000
   エントリー価格帯: 94000-96000
   10%利確: 104500
   損切り: 90000
   ```

3. **確認**
   - ワーカーログに `[PAPER] Would BUY BTCUSDT $2` 等が表示される
   - ダッシュボード → シグナル一覧 にシグナルが表示される
   - ポジション一覧 にアクティブポジションが表示される

4. **価格監視のテスト**
   - 30秒間隔で価格チェックが実行される
   - 損切り価格を下回った場合: `[PAPER] Would SELL` が表示され、ポジションがクローズされる
   - 目標価格に到達した場合: 利確モードなら売却、ピラミッドモードなら追加投資（シミュレート）

5. **手動利確のテスト**
   - ポジション一覧 で「手動利確」ボタンをクリック
   - ポジションがクローズされ、売買履歴に記録される

### パターン B: 手動利確のみのテスト

1. **DB に直接テストポジションを挿入**（任意）
   
   または、Discord シグナルでポジションを作成した後、手動利確ボタンでクローズを試す。

2. **ダッシュボードで手動利確**
   - ポジション一覧 → 手動利確
   - サーバーログに `[PAPER] Would SELL` が表示され、ポジションがクローズされる

---

## 本番移行の切り替え

テスト完了後、`.env.local` から `PAPER_TRADING=true` を**削除**するか、`PAPER_TRADING=false` に変更してください。

```env
# 削除するか、以下に変更
# PAPER_TRADING=false
```

---

## 注意事項

- ペーパー時も **価格取得は実際の Bitget API** を使用します（公開APIのため制限なし）
- シグナル解析・価格チェック・DB の更新は全て本番と同じロジックで動作します
- Bitget スポットはテストネット非対応のため、本番口座で少額テストする場合は `PAPER_TRADING` を削除し、初期投資額を $2 程度に設定してください
