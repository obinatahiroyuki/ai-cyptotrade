# 仮想通貨自動売買システム 要件定義書

## 1. プロジェクト概要

### 1.1 目的
AIエージェント（OpenClaw）を活用した仮想通貨の自動売買システムを構築する。

### 1.2 技術スタック

| 項目 | 技術 | 備考 |
|------|------|------|
| 開発環境 | Cursor | AI支援による開発 |
| AIエージェント | OpenClaw | 取引判断・実行の自動化 |
| ソースコード管理 | GitHub | バージョン管理・CI/CD |
| ホスティング・デプロイ | Vercel | フロントエンド・API |
| データベース | Turso | 取引履歴・設定の永続化 |
| フレームワーク | Next.js | 既存プロジェクトベース |

---

## 2. 実現可能性の評価

### 2.1 結論：**可能（アーキテクチャの工夫が必要）**

### 2.2 各技術の適合性

#### OpenClaw × 本プロジェクト
- **適合**: OpenClawは100以上の取引所（Binance, OKX, Bitget, Hyperliquid等）に対応
- **リスク管理**: ポジション上限2%、日次/週次損失制限、ストップロスが組み込み済み
- **AIモデル**: Claude Opus 4.5、サブ秒級の実行速度
- **戦略**: モメンタム、DCA、グリッド、平均回帰、センチメント分析、アービトラージに対応

#### Vercel × OpenClaw
- **注意点**: Vercelはサーバーレス向け。OpenClawは24/7稼働の長時間プロセス
- **対応策**:
  - **Vercel Sandbox**: マイクロVMでOpenClawを独立稼働可能（ポート18789でHTTPS公開）
  - **推奨**: ダッシュボード・APIはVercel、エージェント本体は別プラットフォーム（Cloudflare Workers、AWS ECS、VPS等）で稼働させる構成
- **本プロジェクト**: Next.js製の管理UI・設定画面はVercelに最適

#### Turso × 本プロジェクト
- **適合**: SQLite互換のエッジDB。取引履歴、ポジション、ユーザー設定の保存に適する
- **利点**: 低レイテンシ、サーバーレスとの相性が良い

#### GitHub
- **適合**: 標準的なソース管理・CI/CDパイプライン構築に問題なし

## 3. システムアーキテクチャ

### 3.1 採用設計：責務分離・プラットフォーム非依存

**方針**: ダッシュボードとエージェントを完全に分離し、Tursoを唯一の状態管理とする。エージェントはどこでも稼働可能とし、将来的なプラットフォーム移行を容易にする。

```
┌─────────────────────────────────────────────────────────────────┐
│                        ユーザー                                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Vercel (Next.js)                                                │
│  ├── ダッシュボード（ポジション・損益・チャート表示）                │
│  ├── 設定画面（取引所API連携、リスクパラメータ）                    │
│  └── API Routes（Tursoへの読み書き、エージェント接続先の参照）      │
└─────────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  Turso          │ │  OpenClaw        │ │  取引所API       │
│  (唯一の状態)   │ │  (AIエージェント) │ │  (Binance等)     │
│  取引履歴       │ │  市場分析・売買   │ │                  │
│  ポジション     │ │  実行            │ │                  │
│  設定           │ │  ※どこでも稼働   │ │                  │
└─────────────────┘ └─────────────────┘ └─────────────────┘
         ▲                     │
         └─────────────────────┘
           設定・状態の読み書き
```

### 3.2 設計原則（移行容易性の確保）

| 原則 | 実装方針 |
|------|----------|
| **責務の分離** | ダッシュボード＝表示・設定、エージェント＝取引実行のみ。直接結合しない |
| **状態の一元化** | ポジション・設定・履歴はすべてTursoに保存。エージェントはステートレス |
| **接続先の外部化** | エージェントのURL（`OPENCLAW_GATEWAY_URL`）は環境変数で管理 |
| **通信の標準化** | ダッシュボード↔エージェントはREST API/WebSocket。Vercel固有機能に依存しない |
| **設定のDB経由** | 戦略パラメータ・リスク設定はTurso経由で共有。環境ごとの差分は環境変数のみ |

### 3.3 エージェントの稼働戦略

| フェーズ | 稼働場所 | 目的 |
|----------|----------|------|
| **検証・開発** | Vercel Sandbox | 動作確認、小規模テスト |
| **本番・移行後** | AWS ECS / Cloudflare / VPS 等 | 24/7安定稼働 |

**移行時の作業**: エージェントを停止 → 別プラットフォームにデプロイ → 同じTurso・環境変数を指定 → 再開。ダッシュボードの変更は接続先URLの更新のみ。

## 4. 機能要件

### 4.1 必須機能（MVP）

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-001 | 取引所連携 | APIキー登録・接続テスト（Binance/OKX等） | 高 |
| F-002 | ポジション表示 | 現在の保有ポジション・損益の一覧表示 | 高 |
| F-003 | 取引履歴 | 売買履歴の保存・検索・エクスポート | 高 |
| F-004 | AIエージェント制御 | OpenClawの起動/停止、戦略パラメータ設定 | 高 |
| F-005 | リスク設定 | ポジション上限、日次損失制限、ストップロス | 高 |
| F-006 | 通知 | 取引実行・アラートの通知（Telegram等） | 中 |

### 4.2 拡張機能（フェーズ2）

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| F-007 | バックテスト | 過去データでの戦略検証 | 中 |
| F-008 | マルチ取引所 | 複数取引所の同時運用 | 中 |
| F-009 | カスタム戦略 | ユーザー定義の戦略・シグナル | 低 |
| F-010 | レポート | 月次・週次のパフォーマンスレポート | 低 |

## 5. 非機能要件

### 5.1 セキュリティ
- APIキーは暗号化してTursoに保存
- 環境変数によるシークレット管理（Vercel Environment Variables）
- 取引実行前のHuman-in-the-Loop（大口取引時の承認フロー）

### 5.2 可用性
- ダッシュボード: VercelのSLAに準拠
- エージェント: 24/7稼働を想定（稼働プラットフォームに依存）

### 5.3 パフォーマンス
- ダッシュボード: 初回表示3秒以内
- 取引実行: OpenClawのサブ秒級レスポンスを維持

### 5.4 コンプライアンス
- 利用規約・リスク開示の明示
- 金融商品取引に係る法規制の確認（対象地域による）

## 6. データモデル（Turso）

### 6.1 主要テーブル案

| テーブル | 用途 |
|----------|------|
| `users` | ユーザー情報、認証 |
| `exchange_connections` | 取引所APIキー（暗号化）、接続状態 |
| `positions` | 現在のポジション（通貨、数量、取得価格） |
| `trades` | 取引履歴（売買、価格、手数料、タイムスタンプ） |
| `agent_settings` | OpenClawの戦略・リスクパラメータ |
| `alerts` | 通知履歴、アラート設定 |

### 6.2 環境変数（責務分離のための設定）

| 変数名 | 用途 | 設定場所 |
|--------|------|----------|
| `TURSO_DATABASE_URL` | Turso接続URL | ダッシュボード・エージェント両方 |
| `TURSO_AUTH_TOKEN` | Turso認証トークン | 同上 |
| `OPENCLAW_GATEWAY_URL` | エージェントのゲートウェイURL | ダッシュボード（接続先の切り替え用） |
| `ENCRYPTION_KEY` | APIキー暗号化用 | ダッシュボード |

※エージェント側は取引所APIキーをTursoから取得するか、環境変数で直接受け取るかを実装時に決定する。

## 8. リスク・注意事項

1. **資金リスク**: 自動売買は損失を伴う。テストネットでの運用検証を推奨
2. **API制限**: 取引所のレート制限・API変更に注意
3. **Vercel制約**: エージェントの長時間稼働はSandboxまたは別プラットフォームを検討
4. **法規制**: 地域ごとの仮想通貨取引に関する規制を確認すること

## 9. 本番環境移行チェックリスト

**本番運用に移行する前に、以下を必ず確認・実施すること。**

### 9.1 セキュリティ（必須）

| 項目 | 実施内容 | 理由 |
|------|----------|------|
| **Turso認証トークンの再発行** | `turso db tokens create ai-cyptotrade` で新トークンを発行し、Vercel・エージェントの環境変数を更新。開発中にチャットやログに流出した可能性のあるトークンは使用しない | 漏洩したトークンでDBに不正アクセスされるリスクを排除 |
| **取引所APIキーの再発行** | 本番用に新規APIキーを発行し、開発時使用分は無効化 | 同上 |
| **ENCRYPTION_KEYの変更** | 本番用に新規生成した鍵を使用（開発時と別の値にする） | 暗号化されたAPIキーの復号リスクを排除 |
| **環境変数の確認** | `.env.local` がGitにコミットされていないこと、VercelのEnvironment Variablesに本番用の値が設定されていることを確認 | シークレットの誤公開防止 |

### 9.2 運用・インフラ

| 項目 | 実施内容 |
|------|----------|
| エージェントの稼働場所 | Vercel SandboxからAWS/Cloudflare/VPS等への移行を検討 |
| テストネットでの検証 | 実資金投入前にテストネットで十分に動作確認 |
| 監視・アラート | 異常検知時の通知体制を整備 |
